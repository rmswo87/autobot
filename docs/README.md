# Autobot 문서 디렉토리

## 📚 문서 목록

### 핵심 문서 (필수 읽기)

1. **[DEVELOPMENT_PLAN.md](./DEVELOPMENT_PLAN.md)** ⭐
   - 프로젝트 개발 계획서 (마스터 문서)
   - 프로젝트 구조 설계
   - 모듈별 상세 설계
   - 개발 단계별 체크리스트

2. **[ARCHITECTURE.md](./ARCHITECTURE.md)** ⭐
   - 전체 아키텍처 설계
   - 프론트엔드/백엔드 아키텍처
   - 보안 아키텍처
   - 데이터 플로우

3. **[MODULE_DEPENDENCIES.md](./MODULE_DEPENDENCIES.md)** ⭐
   - 모듈 의존성 맵
   - 의존성 규칙 및 검증
   - 순환 의존성 체크

4. **[PROJECT_STRUCTURE.md](./PROJECT_STRUCTURE.md)**
   - 현재 프로젝트 구조
   - 생성된 파일 목록
   - 경로 별칭 설정

### 설정 및 가이드

5. **[SETUP_GUIDE.md](./SETUP_GUIDE.md)**
   - 프로젝트 초기 설정 가이드
   - Supabase 설정
   - 외부 API 설정
   - 개발 환경 구축

6. **[EXPERT_FEEDBACK.md](./EXPERT_FEEDBACK.md)**
   - 전문가 피드백 및 개선 제안
   - 아키텍처 개선 방안
   - 기능적 개선 제안

---

## 🎯 문서 사용 가이드

### 새로 시작하는 경우
1. `DEVELOPMENT_PLAN.md` 읽기
2. `SETUP_GUIDE.md` 따라하기
3. `PROJECT_STRUCTURE.md` 확인하기

### 모듈 개발 시
1. `DEVELOPMENT_PLAN.md`에서 해당 모듈 설계 확인
2. `MODULE_DEPENDENCIES.md`에서 의존성 확인
3. `ARCHITECTURE.md`에서 아키텍처 패턴 확인

### 문제 해결 시
1. `SETUP_GUIDE.md`의 문제 해결 섹션 확인
2. `MODULE_DEPENDENCIES.md`에서 의존성 문제 확인
3. `EXPERT_FEEDBACK.md`에서 관련 조언 확인

---

## 📝 문서 업데이트 규칙

- 모든 문서는 변경 시 날짜와 버전 업데이트
- 새로운 모듈 추가 시 `MODULE_DEPENDENCIES.md` 업데이트 필수
- 아키텍처 변경 시 `ARCHITECTURE.md` 업데이트 필수
